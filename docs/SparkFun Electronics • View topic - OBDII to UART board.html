Re: OBDII to UART board

by vvmaks » Wed Jun 01, 2011 1:21 pm

When the protocol is set to 6, STN1110 interprets the messages as ISO15765 messages and treats the first data byte as the PCI. What you need to do, is switch to "raw CAN" mode. So first, change the default baud rate for Protocol B (USER1) to 500 kbps:

Code: Select all
ATPP 2D SV 01
ATPP 2D ON
ATZ


After the device resets, simply switch to protocol B, and monitor as usual:

Code: Select all
ATSP B
ATH1
ATD1 // print the DLC, optional
ATMA


I can't tell you which frames are Tx and which ones are Rx, they all look the same on the bus. However, one useful trick is to use filters to isolate the messages of interest. See the "Filtering ST Commands" section of the STN1100 Family Reference and Programming Manual (http://www.scantool.net/scantool/downlo ... 0-frpm.pdf). Also see the "Message Filtering: CAN Protocols" flowchart in Section 8.2, "CAN Protocols".

In a nutshell, you look at which CAN IDs are being sent on the bus, and block the ones you are not interested in (OBDLink allows you to set up multiple filters). For example, to block messages with CAN ID $300:

Code: Select all
STFCB // clear all block filters
STFAP 000,000 // set pass filter to pass all messages
STFAB 300, FFF // block messages with CAN ID $300


If you know the IDs of the messages that you want to pass, you can pass only messages that have CAN ID $760:

Code: Select all
STFCP // clear all pass filters
STFCB // clear all block filters
STFAP 760, FFF // add a filter to pass CAN ID $760


Use the STM command to monitor.
